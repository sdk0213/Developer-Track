# 안드로이드 캐싱 전략
* 기본 지식
  * 접근속도
    * 메모리 > IO
  * LRU (Least Recently Used)
    * 존재한다면 최근 위치로 이동
    * 없다면 최근 위치에 삽입
    * 사이즈를 초과시 가장 오랫동안 참조되지 않는 갑 삭제
    * 예
      * cache = [1, 2, 3, 4, 5]
      * 3을 넣음 : [1, 2, 4, 5, 3]
      * 7을 넣음 : [2, 4, 5, 3, 7]
      * 2는 넣음 : [4, 5, 3, 7, 2]
      *

* IO 캐시

    * /data/data/${PkgName}/cache 에 
    * 또는 외부저장소의 패키지별의 캐시
